About Firera - Q&A

  • Is this for frontend only?
    Mostly. Although Firera's core is just a Functional Rective Programming(FRP) engine that can be used both on backend and frontend, Firera is targeted firstly for managing the UI in browser. That's why an Ozenfant template engine is included in Firera's dist. Also it's handy for communicating with backend etc.
  • So I can write the whole frontend on Firera?
    Yes! While a lot of FRP libraries are focused on specific sides of frontend apps, Firera beleives you can write the whole application as a set of data streams.
  • Is this a library? A framework?..
    It looks like a general purpose JS library. A framework might be written some day on the Firera's base. Also it has a features of a... new high-level language, compiled into JavaScript.
  • What are the key concepts of Firera?
    Firera application consists of interdependent data streams. Unlike most FRP libraries, it has declarative syntax. It promotes pure functions usage, good composability and readability. Data are immutable. As a language, Firera is homoiconic.
  • Where should I use Firera - small, medium or big projects?
    Firera suits for both small and big projects. It can be used as a additional tool
  • What are the cool features of Firera?
    • Declarative approach: do more with less code.
    • Minimalistic, pure, fast HTML templates with Ozenfant.
    • No shared mutable state
    • Laconic and expressive. TodoMVC in 60 LOC of JS code(not including HTML template).
    • Makes you create separate reusable components(in Firera's language "grids")
  • What are the weakpoints?
    • A sophisticated and complex syntax that might look strange and weird
    • Not ready for enterprise
    • Not a silver bullet.
  • Comparing with other libraries...
    • React: less verbose, but almost as precise as React in controlling data flow. Real reactivity, less code. HTML rendering speed: comparable.
    • Cycle.js: common philosophy, different syntax.
    • Vue: a bit similar in templates, component structre. No data mutation, more FRP features.
  • Is it easy to learn Firera?
    Well, in general - no. Moving to Firera from an imperative coding, e.g. with Angular, might be as hard as moving to Angular from jQuery. However, if you are familiar with FRP concept, it would be much easier to understand how Firera works. The most similar library is Cycle.js, though it has an imperative syntax.
  • Is Firera well tested? Ready for production?
    Currently, we have most of code covered with tests. However, the library hadn't been battle-tested in large projects yet.
  • Firera might be interesting for people who...
    like functional programming, reactive programming...
    enjoy declarative approach...
    are familiar with Lisp, Clojure/ClojureScript...
    are tired of HTML+JS salad...
