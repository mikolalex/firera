Getting started with Firera: declarative FRP

Let's start with a simple example: var base = { a: 10, b: 20, c: [(n, m) => n + m, 'a', 'b'] } // Create a Firera app instance var app = Firera({ __root: base }) app.get('c'); // 30 // Here comes a FRP magic app.set('b', 32); app.get('c'); // 42

It's a typical example for introducing functional reactive programming priciples. Here 'a' and 'b' are just values, and 'c' is a computable cell. It means that the value of "c" will always be the sum of "a" and "b", and we don't need to update it manually. There is no proactive intrusion: you cannot change the value of "c" manually, it's changed based on the values of other cells(in this case "a" and "b"). Instead of most other JS FRP libraries like Rx or Most, which has method chains, Firera's approach is declarative.

In Firera language, "app_base" variable is called plain base, "a", "b", "c" are cells, and the expression [(n, m) => n + m, 'a', 'b'] is called formula.
